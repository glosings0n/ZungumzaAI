**Zungumza.AI** bridges AI research and humanitarian response in Eastern DRC.
We are engineering offline, multimodal Artificial Intelligence models
(ASR/TTS) tailored for low-resource African languages, starting with
Kilendu.

Our mission is to power interactive, mother-tongue educational
applications for displaced children in conflict zones, helping combat
both the digital divide and learning loss.

## About the Project

Protracted conflicts in Eastern Africa have caused large-scale
displacement. In many Internally Displaced Persons (IDP) camps,
children have limited access to physical schools. At the same time,
many EdTech systems and modern AI tools systematically exclude
vernacular languages.

Zungumza.AI addresses this gap by building a scientifically grounded
linguistic corpus for Kilendu and developing edge-ready AI models.
This enables educational applications to run fully offline on rugged,
solar-powered tablets in emergency learning hubs.

## Key Features

- Low-resource NLP: Custom Automatic Speech Recognition (ASR) and
	Text-to-Speech (TTS) for underrepresented dialects.
- Edge-AI deployment: Optimized neural networks designed to run
	offline on low-end hardware.
- Multimodal data collection: Tools and protocols for participatory
	field data gathering and linguistic annotation.
- EdTech integration: APIs for connecting AI services to Flutter-based
	educational games and virtual tutors.

## Tech Stack

- Machine Learning: TensorFlow/PyTorch, Hugging Face Transformers
	(for example wav2vec 2.0).
- Application Development: Flutter and Dart.
- Cloud and Infrastructure: Google Cloud Platform (GPU training),
	Firebase.
- Data Annotation: Open-source linguistic annotation tools.

## How to Contribute

Zungumza.AI is a community-driven open-source project. We welcome
contributions from ML engineers, Flutter developers, linguists, and
humanitarian technology contributors.

For full contribution guidelines, see [CONTRIBUTING.md](CONTRIBUTING.md).

Quick flow:

```bash
# 1) Fork the project on GitHub, then clone your fork
git clone https://github.com/YOUR_USERNAME/ZungumzaAI.git
cd ZungumzaAI

# 2) Create a feature branch
git checkout -b feature/amazing-feature

# 3) Commit and push your changes
git add .
git commit -m "feat: add amazing feature"
git push origin feature/amazing-feature
```

Then open a Pull Request against this repository with a clear
description of your changes.
