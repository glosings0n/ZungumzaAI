# Contributing to Zungumza.AI

Thank you for your interest in contributing to Zungumza.AI.
We welcome contributions from engineers, researchers, linguists,
educators, and humanitarian technology practitioners.

## Code of Conduct

By participating, you agree to be respectful, inclusive, and
constructive in all interactions.

## Ways to Contribute

- Report bugs and propose improvements via GitHub issues.
- Improve documentation and onboarding content.
- Contribute ML, ASR, and TTS experiments.
- Support data tooling, annotation workflows, and quality checks.
- Build integration components for educational applications.

## Getting Started

1. Fork the repository.
2. Clone your fork locally.
3. Create a branch from `main`.
4. Make focused changes with clear commit messages.
5. Open a pull request.

## Development Flow

```bash
git clone https://github.com/glosings0n/ZungumzaAI.git
cd ZungumzaAI
git checkout -b feature/short-description
```

When your changes are ready:

```bash
git add .
git commit -m "type: concise summary"
git push origin feature/short-description
```

Then open a pull request to `main` in this repository.

## Pull Request Guidelines

- Keep PRs focused and reasonably small.
- Include a clear problem statement and what changed.
- Link related issues when applicable.
- Add tests or validation notes when relevant.
- Update documentation for user-facing changes.

## Commit Message Convention

Use a simple conventional style when possible:

- `feat:` new functionality
- `fix:` bug fix
- `docs:` documentation changes
- `refactor:` internal improvements
- `test:` tests added or updated
- `chore:` maintenance updates

Example:

```text
feat: add baseline kilendu asr preprocessing script
```

## Issue Reporting

When filing an issue, include:

- Expected behavior
- Actual behavior
- Steps to reproduce
- Logs or screenshots (if available)
- Environment details

## Data and Ethics

Given the humanitarian context, contributors should:

- Avoid uploading sensitive personal data.
- Respect community consent and data governance principles.
- Document data provenance and annotation assumptions.

## Questions

If you are unsure where to start, open an issue with context about
what you would like to contribute, and maintainers can guide you.
